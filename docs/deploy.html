Why it just ships

The app is self-contained and deploy-ready: the 68 lessons are baked into the image, it honors $PORT, it converts host postgres:// URLs to asyncpg automatically, and it falls back to JSON embeddings if a host's Postgres lacks pgvector. No code changes are needed to go live.

The free stack at a glance

1 · Render

Hosts the Docker web service. A one-click Blueprint reads render.yaml and provisions the service plus a free Postgres. The free plan sleeps after 15 min idle.

2 · Groq

Powers the AI tutor on a free tier with no credit card. Roughly 30 requests/min, plenty for a single learner. Swap in Claude or any OpenAI-compatible host later.

3 · Postgres

Render Postgres (free, ~30-day) for a quick demo, or Neon (free, persistent, pgvector) for longevity. Set DATABASE_URL and you are done.

Step 1 — get a free Groq key

Go to console.groq.com, open API Keys, and click Create. No credit card is required. The free tier allows about 30 requests per minute, which is plenty for one learner.

# the key looks like this — keep it secret, paste it into the host's dashboard
GROQ_API_KEY=gsk_...

Step 2 — deploy the app on Render

Two paths. Option A is the one-click Blueprint and is the easiest. Option B is the manual Docker web service if you want full control over each setting.

Option A — one-click Blueprint (easiest)

The repo ships a render.yaml that provisions the web service and a free Postgres for you, wiring DATABASE_URL in automatically and generating JWT_SECRET.

StepAction
1Push the learning-app/ folder to its own GitHub repo.
2In the Render dashboard, choose New → Blueprint and select the repo. Render reads render.yaml and provisions the web service and a free Postgres.
3When prompted, paste your GROQ_API_KEY (it is kept out of git via sync: false).
4Open the URL Render assigns and click Register to create your account. The demo seed account is intentionally not created on the hosted Postgres database.
What the Blueprint sets for you

From render.yaml: a Docker web service on the free plan with health check /api/health, a generated JWT_SECRET, LLM_PROVIDER=groq, the tutor and judge models (llama-3.3-70b-versatile / llama-3.1-8b-instant), a free Postgres 16 named mastery-db, and DATABASE_URL bound to that database's connection string. You only supply GROQ_API_KEY.

Option B — manual Docker web service

  1. In Render, choose New → Web Service, connect the repo, and set Runtime: Docker.
  2. Add the environment variables below.
  3. Deploy.
# minimum env vars for a manual Render web service
LLM_PROVIDER=groq
GROQ_API_KEY=gsk_...
JWT_SECRET=# any 32+ char random string
DATABASE_URL=# from your database, see Step 3
Free service sleeps

The free web service sleeps after 15 minutes idle, and the first request after that takes about a minute to wake the container. This is normal for free hosting and does not affect your data.

Step 3 — the database

Pick one. Render Postgres is the zero-effort demo option; Neon is the better choice if you want the data to stick around and to use pgvector for semantic features.

Render Postgres (free)

Provisioned automatically by the Blueprint, so it is the easiest path. Note that the free instance expires roughly 30 days after creation; recreate or upgrade it when it does.

free ~30-day auto-wired

Neon (free, persistent, pgvector)

Recommended for longevity. Create a project at neon.tech, copy the connection string, and set DATABASE_URL plus PG_SSL=true on Render. Neon supports pgvector, so semantic features light up once you add embeddings.

free persistent pgvector 
# Neon: set these two on Render
DATABASE_URL=postgresql+asyncpg://USER:PASSWORD@HOST/dbname
PG_SSL=true
Do not use SQLite in production

SQLite is not suitable on Render or any ephemeral host: the disk is wiped on every deploy and on sleep, so your accounts and progress would reset. SQLite is for zero-config local development only. Use Postgres in production.

Step 4 — verify

Open /api/meta on your deployed URL. You should see your lesson and question counts, the SRS engine, and the active LLM provider. Then sign in and try the AI Tutor.

# GET https://your-app.onrender.com/api/meta
{
  "srs_engine": "FSRS-6",
  "lessons": 68,
  "questions": 110,
  "ai": { "provider": "groq" }
}

Environment variables

Everything the app reads at startup. The Blueprint sets most of these for you; for a manual deploy, supply at least GROQ_API_KEY, JWT_SECRET, and DATABASE_URL.

VariableExampleWhat it does
LLM_PROVIDERgroqSelects the tutor backend: groq, anthropic, or auto to auto-detect from whichever key is present.
GROQ_API_KEYgsk_...Groq key from console.groq.com. Also reused as the key when pointing at an OpenAI-compatible host.
GROQ_MODELllama-3.3-70b-versatileThe tutor model used for teaching, deepening, and mock interviews.
GROQ_JUDGE_MODELllama-3.1-8b-instantThe smaller, cheaper model used to grade answers (the judge half of the tutor/judge split).
ANTHROPIC_API_KEYsk-ant-...Claude key, used when LLM_PROVIDER=anthropic (Opus 4.8 tutor, Haiku judge).
OPENAI_BASE_URLhttps://your-host/v1Points the client at any OpenAI-compatible endpoint (OpenAI, Together, OpenRouter, local Ollama).
JWT_SECRETany 32+ char stringSigns the JWT auth tokens. Keep it secret and stable; the Blueprint generates one.
DATABASE_URLpostgresql+asyncpg://...The Postgres connection string. Host postgres:// URLs are normalized to asyncpg automatically.
PG_SSLtrueEnables SSL on the database connection. Set true for Neon and most managed Postgres hosts.

Switching the LLM provider

The tutor is provider-agnostic and degrades gracefully with no key (the rest of the app still works). Bring your own key and set LLM_PROVIDER with the matching variables.

Groq (free)

The default for hosting. Set LLM_PROVIDER=groq and GROQ_API_KEY. Uses llama-3.3-70b-versatile as tutor and llama-3.1-8b-instant as judge.

Anthropic (Claude)

Set LLM_PROVIDER=anthropic and ANTHROPIC_API_KEY. Uses Opus 4.8 as tutor and Haiku as judge, the same cost-aware split.

Any OpenAI-compatible

For OpenAI, Together, OpenRouter, or a local Ollama: keep LLM_PROVIDER=groq, set GROQ_API_KEY to that host's key, and point OPENAI_BASE_URL at it.

# Groq — FREE, no credit card (https://console.groq.com)
LLM_PROVIDER=groq
GROQ_API_KEY=gsk_...
# …or Claude
LLM_PROVIDER=anthropic
ANTHROPIC_API_KEY=sk-ant-...
# …or ANY OpenAI-compatible host (OpenAI, Together, OpenRouter, local Ollama)
LLM_PROVIDER=groq
GROQ_API_KEY=<that host's key>
OPENAI_BASE_URL=https://your-host/v1

Alternatives

Fly.io

Generous, but now requires a card on file. Run the interactive login first, then launch (it detects the Dockerfile), set your secrets, attach Postgres, and deploy.

# run interactive login first
flyctl auth login

flyctl launch                # detects the Dockerfile
fly secrets set GROQ_API_KEY=... LLM_PROVIDER=groq JWT_SECRET=...
fly postgres create
fly postgres attach
fly deploy

Railway / Northflank

A similar Docker plus Postgres flow. Provision the service from the Dockerfile and set the same environment variables listed above.

Local production stack

To run the full stack on your own machine, Docker Compose brings up the API plus Postgres (with pgvector) plus Redis.

# API + Postgres(pgvector) + Redis on your machine
GROQ_API_KEY=gsk_... docker compose up --build
