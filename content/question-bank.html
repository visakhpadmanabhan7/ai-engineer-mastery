Foundations & practical patterns

Full deep-dive: Notion · Foundations · Lessons: Module 1

  • Explain how an LLM generates text to a smart non-engineer.
    Key: next-token prediction over a vocabulary, looped; context steers the distribution.
  • What is an agent, and how is it different from a chain?
    Key: agent = LLM has autonomy over control flow (picks next step + tool in a loop); chain = fixed, dev-wired pipeline.
  • When should you NOT use an agent?
    Key: scoped/deterministic task, tight latency, single-prompt-expressible, safety-critical. Knowing when not to is the senior signal.
  • What is agentic retrieval vs a fixed retrieval step?
    Key: retrieval as a tool the LLM chooses to call, with crafted queries and multiple iterations.
  • Prompt engineering vs context engineering?
    Key: prompt = static contract (persona/format, written once); context = the dynamic per-turn supply chain (chunks, history, tool results, order). Most prod failures are context failures.
  • What makes a good tool definition?
    Key: action-verb name, "use this when / do NOT use when" docstring, tight typed params, return description. Negative scoping matters.
  • Where should a rule live: system prompt / tool / few-shot / retrieval / metadata?
    Key: external data/action→tool; persona/format→system; hard-to-describe pattern→few-shot; filter-by-attribute→metadata; multi-tenant policy→retrieval, not the prompt.
  • Foundation models vs traditional ML, when does each win?
    Key: FM for language-rich/ambiguous/multi-task + fast iteration; traditional for narrow, labeled, low-latency, regulated, cheap-at-scale.
  • How is AI Engineering different from ML Engineering?
    Key: ML trains custom models from data (data-centric, slow cycle); AI eng builds on foundation models via API (prompt-centric, fast cycle).
  • Pretraining vs fine-tuning vs prompting, how do you choose?
    Key: effort/cost ladder prompting < fine-tuning < pretraining. Exhaust prompting + RAG + few-shot first; pretraining almost never for a product team.

Prompting & context engineering

Lessons: Module 2

  • Why does structured output matter and how do you enforce it?
    Key: downstream code parses it; enforce via tool/function calling (schema-enforced) or Pydantic > JSON mode > prompt; constrained decoding is strongest.
  • When does structured output break?
    Key: over-generated optional fields, similar enums, truncated long JSON, deep nesting (>3 levels). Pair with post-validation.
  • What is the 3-layer constraint model for outputs?
    Key: L1 output format (tool calling) / L2 schema (Pydantic) / L3 semantic ("does this date make sense?").
  • Few-shot vs zero-shot vs chain-of-thought, when each?
    Key: zero-shot for simple; few-shot to pin format/edge cases; CoT for multi-step reasoning (costs tokens + latency).
  • What is the "sandwich" pattern?
    Key: cheap deterministic model on the input, expensive FM in the middle, cheap classifier on the output.

RAG & retrieval core

Full deep-dive: Notion · RAG · Lessons: Module 3

  • What is RAG and why use it instead of fine-tuning?
    Key: retrieves context at query time; wins on dynamic knowledge, provenance/citations, no retrain.
  • Walk through a vanilla RAG pipeline.
    Key: ingest → chunk → embed → index → retrieve → generate.
  • What is the most common RAG failure mode?
    Key: retrieval (~70-75% of bad answers): bad chunking, bad embeddings, no reranker, wrong top-k.
  • Compare fixed / semantic / sentence-window / parent-child chunking.
    Key: fixed = N-token+overlap (cheap, breaks boundaries); semantic = similarity splits; sentence-window = index sentence, expand at retrieval; parent-child = index small, return parent. Start parent-child.
  • How do you choose chunk size and overlap?
    Key: embedder max tokens + question granularity + context budget; start 256-512 tokens, 10-20% overlap.
  • What is BM25 and why does it still matter in 2026?
    Key: probabilistic bag-of-words (TF-IDF + length norm + TF saturation); dominates exact-match (codes, names, acronyms).
  • Why ANN instead of brute-force cosine?
    Key: brute force is O(N·d) and hits a cliff; ANN (HNSW/IVF/PQ) trades a little recall for big speed; HNSW default.
  • What is hybrid search and how do you fuse scores?
    Key: BM25 + dense in parallel, merge via RRF (rank-only, scale-free) or weighted fusion; RRF avoids score-scale mismatch.
  • What is reranking and where does it sit?
    Key: rescore top 50-100 with a cross-encoder (joint query,chunk); the precision booster before generation.
  • Bi-encoder vs cross-encoder vs ColBERT?
    Key: bi-encoder scales retrieval (precompute); cross-encoder for small-N rerank (accurate, slow); late-interaction in the middle.
  • pgvector vs Pinecone vs Qdrant vs Weaviate vs Milvus?
    Key: pgvector = vectors next to relational (≤~10M); Pinecone = managed scale; Qdrant/Weaviate = first-class filter/delete; Milvus = huge scale.
  • What are recall@k, MRR, NDCG?
    Key: recall@k = did we retrieve it at all (most important); MRR = 1/rank of first relevant; NDCG = graded, log-discounted.
  • How do you evaluate generation, not just retrieval?
    Key: RAGAS-style LLM-as-judge: faithfulness (trust most), answer relevancy, contextual precision/recall.
  • How do you prove a retrieval change is actually better?
    Key: offline golden set → shadow/interleaving → online per-user A/B with guardrail metrics.
  • Multi-tenant RAG isolation?
    Key: metadata filter → per-tenant namespace → per-tenant index; never drop the tenant filter (data-layer constraint, not LLM filtering).
  • What is query rewriting / HyDE / multi-query?
    Key: transform the query before retrieval (decomposition, step-back, HyDE); help ambiguous/multi-hop, can hurt precise queries.
  • What is GraphRAG / RAPTOR / CRAG, when worth it?
    Key: GraphRAG = entity-relation KG + summaries for global/multi-hop; RAPTOR = recursive summary tree for multi-doc synthesis; CRAG = grade chunks, drop irrelevant, fall back.
  • How do you cut RAG cost ~30% without quality loss?
    Key: embed once, cache reranks, route, lower top-k, smaller generator, prompt caching. (Your real story: 11+ experiments, 37% cut, 2% drop.)

Agents core

Full deep-dive: Notion · Agents · Lessons: Module 4

  • What is the ReAct loop and why still the single-agent default?
    Key: interleaves Thought → Action → Observation; simple, debuggable from the trace; good for a handful of tool calls.
  • Plan-Execute vs ReAct, when does Plan-Execute win?
    Key: >6 mostly-parallel steps knowable upfront; planner emits a DAG, executor runs, re-planner inspects.
  • Supervisor vs Swarm vs Hierarchical topology?
    Key: supervisor = one routing LLM (default, debuggable, slower); swarm = direct handoffs (faster, harder to trace); hierarchical for >20 agents.
  • Large tool space (50+) without wrong picks?
    Key: tool retrieval (embed descriptions, top 5-10), categorical routing, or supervisor; ~10-20 tools is the sweet spot.
  • What is a hallucinated tool call and how do you prevent it?
    Key: nonexistent name or schema-violating args; prevent via structured tool calling, registry validation, tight typed args, self-correction (bad args > bad names).
  • What is idempotency in tool design and why care?
    Key: agents retry loops; create/charge/delete firing twice is dangerous; idempotency keys are non-negotiable.
  • How do you implement an approval gate for high-risk actions?
    Key: classify read vs write; framework interrupt (LangGraph) pauses on the tool; draft action → human approves → execute.
  • LangChain vs LangGraph, when does the upgrade pay off?
    Key: LangChain = integrations/chains; LangGraph = stateful graph runtime (cycles, interrupts, checkpoints); upgrade when control flow gets complex.
  • Agents are non-deterministic, how do you sleep at night?
    Key: five layers, constrain the action space, tracing on every call, eval (trajectory + output), guardrails/approval, canary + rollback.
  • How do you evaluate an agent?
    Key: final-answer correctness (LLM-judge) + tool-call accuracy + trajectory eval + online/safety; start with the first two.
  • Right answer but wrong tool, bug or feature?
    Key: bug; right answer on a wrong trajectory is luck and breaks under data shift.
  • Short-term vs long-term vs episodic memory?
    Key: short = working context (pruned); long = durable user facts (vector/KV by user_id); episodic = summaries of past sessions.
  • What is MCP vs A2A?
    Key: MCP = standard for agent-to-tool/data; A2A = agent-to-agent across trust boundaries; internal handoffs are just function calls / graph nodes.

Evaluation & MLOps core

Full deep-dive: Notion · Evaluation & MLOps · Lessons: Module 5, Module 7

  • Offline vs online evaluation, when each?
    Key: offline = fixed dataset vs expected (golden/replay/synthetic); online = live signals (thumbs-down, escalation). Use both.
  • How do you build a golden set that does not rot?
    Key: seed by intent, mine prod traces (low-confidence + bug reports), tag intent/difficulty/last-reviewed, refresh quarterly.
  • What is LLM-as-judge and where does it break?
    Key: scales human eval cheaply; biases, position, verbosity, self-preference; calibrate vs humans (kappa/Spearman ≥ ~0.6).
  • How do you build a stable judge rubric?
    Key: numbered scale with explicit per-level anchors, 1-2 in-context examples per level, force reasoning before the score.
  • How do you mitigate judge biases?
    Key: randomize + average both orders (position), normalize length (verbosity), use a different model family (self-preference).
  • When is LLM-as-judge the wrong tool?
    Key: deterministic ground truth (math/code/structured → programmatic), expert domains (→ human), cost-prohibitive scale.
  • Output eval vs trajectory eval?
    Key: output scores the final response; trajectory scores the path (right tools, right order, recovered, policy followed).
  • What does a production tracing pipeline look like?
    Key: span tree per request (LLM/tool/retrieval spans with tokens + cost + latency each), OTel/OTLP, LangSmith/Langfuse view.
  • What do you log vs NOT log per request?
    Key: log prompt version, model, temp, tokens, latency, tool calls + args, doc IDs, hashed user/session, cost; never raw PII.
  • What is shadow mode catching that offline eval misses?
    Key: real input distribution, real tool latency/failures, real concurrency/rate limits.
  • How do you cut LLM spend ~50% without dropping quality?
    Key: prompt caching (biggest lever), semantic cache, model routing, shorter context, batch.
  • What SLOs would you set for an LLM endpoint?
    Key: latency p95/p99, error + refusal rate, cost-per-request budget, faithfulness floor on a rolling sample.
  • Walk through shadow → canary → full rollout.
    Key: shadow (parallel, no serve) → canary 1→5→100% → full; eval-gated, rollback ready.

Fine-tuning & adaptation

Lessons: Module 6

  • Prompting vs RAG vs fine-tuning, how do you choose?
    Key: fine-tune changes form/style/latency/cost, not facts; RAG supplies fresh facts; exhaust prompting first.
  • What is LoRA and why is it efficient?
    Key: freeze base weights, train small low-rank adapter matrices; ~99% fewer trainable params, swappable adapters.
  • What is QLoRA?
    Key: LoRA on a 4-bit quantized base; fits big-model fine-tuning on one GPU.
  • What is catastrophic forgetting and how do you avoid it?
    Key: over-tuning erodes general ability; low LR, few epochs, mixed data, adapters.
  • What is DPO vs RLHF?
    Key: DPO optimizes directly on preference pairs (no separate reward model or RL loop); simpler, stable.

LLMOps, serving & optimization

Lessons: Module 7

  • What inference-time optimizations exist and when each?
    Key: prompt caching → KV caching → continuous batching → quantization → speculative decoding.
  • What is KV caching and when does it matter most?
    Key: cache per-layer key/value so each new token attends without recompute; biggest win on long shared prefixes / agent loops.
  • What is continuous batching?
    Key: dynamically add/remove sequences in a serving batch (vLLM) for 2-4x throughput vs static batching.
  • Explain speculative decoding to a skeptic.
    Key: small draft model proposes N tokens, big model verifies in one parallel pass, accepts the matching prefix; same distribution, faster.
  • What is semantic caching and where is it dangerous?
    Key: embed query, return a near-duplicate's cached answer above a cosine threshold; dangerous for personalized/time-sensitive/high-stakes.
  • How do you design a rate limiter for an LLM API client?
    Key: token bucket (refill rate r, capacity = burst) client-side + server-side; atomic check-decrement (Redis Lua).
  • How do you handle 429s from a provider?
    Key: respect Retry-After, exponential backoff + jitter, cap attempts, only retry idempotent calls.
  • What is a circuit breaker and its states?
    Key: Closed (allow, count failures) → Open (fail fast, cooldown) → Half-open (one probe; success closes).

Python & backend

Full deep-dive: Notion · Python & Backend

  • Explain the GIL at a senior level.
    Key: one thread runs bytecode at a time; released during blocking I/O + C extensions; I/O→asyncio, CPU→multiprocessing (3.13 has experimental free-threading).
  • async/await mental model?
    Key: async def returns a coroutine; await yields to the loop; cooperative, a non-awaiting coroutine starves all.
  • Can you have a race condition in single-threaded async?
    Key: yes, any await is a yield point; read-await-write loses updates. Use asyncio.Lock or atomic store ops.
  • FastAPI vs Flask vs Django, pick each?
    Key: FastAPI = async APIs/ML/LLM/OpenAPI (default); Flask = tiny sync; Django = CRUD/admin/ORM.
  • Sync route vs async route in FastAPI?
    Key: async def when all deps are async; def (threadpool) for blocking. Never call requests.get in an async route.
  • Fan out N LLM calls with bounded concurrency?
    Key: asyncio.Semaphore + gather(return_exceptions=True), or TaskGroup (3.11+).
  • SQL vs NoSQL, your default?
    Key: Postgres (relational + JSONB + FTS + pgvector) unless a specific reason; never await an LLM inside a DB transaction.
  • Idempotency keys for a retried POST?
    Key: client sends an Idempotency-Key UUID; server stores (key, response, TTL) and replays the cached result.
  • Cursor vs offset pagination?
    Key: cursor (opaque (sort_key, id) token) for production; offset breaks under inserts and is slow on big tables.

System design (general + AI)

Full deep-dive: Notion · System Design · Lessons: Modules 8–10

  • Walk through your system design methodology.
    Key: clarify (func + non-func: scale, latency, consistency, tenancy) → estimate (QPS, storage) → high-level arch → data model → deep-dive the bottleneck → trade-offs (name what you give up) → ops.
  • Design a URL shortener.
    Key: base62 ~7 chars (~3.5T keyspace), KV store, 301 vs 302 trade-off, async click analytics.
  • Design a rate limiter.
    Key: token bucket / sliding window; Redis atomic; per-user + global; distributed coordination.
  • Explain the CAP theorem in practice.
    Key: under partition choose consistency or availability; most systems tune within CP/AP (PACELC adds the no-partition latency tradeoff).
  • What is sharding and what breaks?
    Key: partition by key across nodes; cross-shard joins/transactions and hot keys are the pain; pick the shard key carefully.
  • Design a production RAG assistant.
    Key: ingestion (parse, parent-child chunk, embed) → hybrid retrieval + rerank + tenant filters → grounded generation with citations → faithfulness eval + observability + cost controls. Touch the 11-point AI checklist.
  • Design a multi-tenant LLM / agent platform.
    Key: tool registry, per-tenant scoping in code, prompt/version mgmt, eval + tracing, cost budgets, sandboxed skills with human review for high-risk scopes.
  • Design an agent marketplace / Skill Store.
    Key: signed manifests, sandboxed per-invocation credentials, static analysis + eval + human review for risky scopes, circuit breakers per skill. (You shipped this in MaiQ.)
  • Design real-time LLM inference with cost optimization.
    Key: continuous batching (2-4x), KV/prefix caching (50%+ on agent loops), quantization, routing, spot + on-demand.
  • Capacity numbers to know.
    Key: 1536-dim float32 ≈ 6 KB (~1.5 KB int8); frontier model ~$5/1M in, ~$15/1M out; aim ~90% cache-hit on hot reads.

Classic system design problems (System Design Primer)

The canonical web-scale problems from The System Design Primer (Donne Martin, MIT). Drive the seven-step framework on each; the keys are one-line solution sketches, not full designs.

  • Design Pastebin.com (or Bit.ly).
    Key: store text/URL under a base62 id (counter or hash); read is an O(1) key lookup, cache-heavy with TTL/expiry; large pastes in object storage, metadata in DB; CDN for hot reads; 301 vs 302 for links.
  • Design the Twitter timeline and search.
    Key: fan-out on write (precompute follower timelines) for fast reads with write amplification; pull for celebrities; hybrid in practice; timeline cache in Redis lists; search via inverted index; eventual consistency is fine.
  • Design a web crawler.
    Key: URL frontier (priority queue) + seen-set (bloom filter) for dedup; politeness (per-domain rate limit, robots.txt); fetch → parse → extract links → enqueue; distributed workers; re-crawl by freshness.
  • Design Mint.com.
    Key: async sync of transactions from bank APIs via queues + workers; idempotent sync; categorize (rules/ML); per-user ledger; budgets + alerts; encrypt sensitive data; eventual consistency on balances.
  • Design the data structures for a social network.
    Key: adjacency lists per user (following / followers), sharded KV or graph DB; BFS for degrees-of-separation and friend suggestions; denormalize and cache hot edges.
  • Design a key-value store for a search engine.
    Key: partition by key with consistent hashing; N-replica quorum (R+W>N); LSM-tree storage + per-SSTable bloom filters for write throughput; versioning / conflict resolution; gossip membership (Dynamo-style).
  • Design Amazon's sales rank by category.
    Key: high-write order events → stream (Kafka) → windowed batch/stream aggregation per category → precomputed ranked lists in a fast store; MapReduce/Spark for history; eventual consistency; cache top-N.
  • Design a system that scales to millions of users on AWS.
    Key: incremental playbook: single box → split web/DB → LB + stateless app fleet → read replicas + cache → CDN for static → shard / move hot data to NoSQL → async queues for spiky work → multi-AZ. Scale the bottleneck the metrics show.

Live coding patterns

Full deep-dive: Notion · Live Coding · Practice: every Lab day

  • Implement a minimal ReAct agent with one tool.
    Key: Pydantic-validated JSON step, max_steps guard, feed parse errors back into the next prompt.
  • Write a recursive character chunker with overlap.
    Key: separator priority \n\n → \n → ". " → " "; carry overlap tokens between chunks.
  • Implement hybrid retrieval via RRF.
    Key: score = sum of 1/(k+rank), k=60, rank-only so it is scale-free across BM25 and dense.
  • Write an LLM-as-judge scorer.
    Key: numbered rubric, JSON out, self-consistency by sampling + averaging, drop malformed outputs.
  • Implement a TTL + LRU cache.
    Key: OrderedDict, lazy expiry, move_to_end on read, popitem(last=False) to evict oldest.
  • Top-k embeddings by cosine.
    Key: L2-normalize then dot product; np.argpartition is O(N), beats argsort for top-k.
  • Rate-limited async LLM batch.
    Key: Semaphore + token-bucket sleep; collect results by index to preserve order.
  • Parse partial JSON from a stream.
    Key: track open brackets/strings, synthesize closers; SSE blocks split on blank lines, watch the [DONE] sentinel.
Live-coding reflexes

Ask 2-3 clarifying questions first; name the trade-off at every fork; reach for Pydantic reflexively; assert schema and length on LLM outputs, not exact strings; backoff with jitter; leave 5 minutes for one happy-path + one edge-case test.

Behavioral & STAR

Full deep-dive: Notion · Behavioral & STAR · Lessons: Module 11

  • Walk me through your career, why AI engineering, why now?
    Key: the arc to production AI; the MaiQ shipping + evaluation focus; concrete and forward-looking.
  • Tell me about an AI system you shipped, what broke first?
    Key: STAR, Action-heavy (50-60%), name the option you ruled out, quantify, end with a learning.
  • A model underperformed in production, what did you do?
    Key: follow the data, isolate retrieval vs generation, eval-driven fix, measured result.
  • How do you decide a model is good enough to ship?
    Key: offline eval gates + online guardrails + a quality floor; risk vs value; rollback ready.
  • A hard reliability trade-off (latency/cost/quality)?
    Key: a real decision (rerank depth vs latency, model routing), what you measured, what you gave up.
  • Biggest professional failure?
    Key: real, owned, with the lesson and what you changed; no fake-humility ("I work too hard").
STAR proportions

Situation/Task ~25%, Action 50-60% (the signal: reasoning, trade-offs, the ruled-out option), Result 15-25% (a metric + a learning). Prepare 6-8 stories re-angled, not 30. Have these five ready: shipped-to-prod, failure-with-learning, conflict, ambiguous-decision, project-led.

Questions to ask back

Full deep-dive: Notion · Questions to Ask Back

  • Hiring manager.
    Ask: what does success look like at 30/60/90 days? What is the first hard problem, and why is it still unsolved? 12-month headcount plan?
  • Peer engineer (the truth round).
    Ask: how do you measure model/agent quality in prod and catch regressions before users? Most painful tech debt + the actual plan? Worst incident this year?
  • CTO / senior leader.
    Ask: the AI strategy in one sentence and the most important bet, and what would prove it wrong? Where is the eng org structurally weak?
  • Founder (seed / Series A).
    Ask: runway and burn? The anti-pitch: why should I NOT join? Biggest mistake running the company and what changed?
  • Best closer.
    Ask: "Any reservations about my fit I could address right now?" Then confirm next steps, timeline, and who else is in the loop.
Avoid

Generic ("what does the company do?"), premature (salary in the first 5 minutes), or loaded ("is there a lot of politics?"). Anchor every question to a specific recent event, person, or decision.
